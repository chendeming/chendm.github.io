Why do I think of using github pages, because I want more people to see the technical related articles I wrote, and to be able to give some opinions, before writing some openclaw use notes, so in the openclaw forum, but found that there are not many people to see, so early what are the good personal blogs, deepseek recommends github pages, which is free, so we started looking for some stuff on the web and started setting it up.
The correct path is as follows:
1. Learn how github pages works from deepseek;
2. Find a beginner tutorial video on youtube.
3. According to the requirements of the video, fork a good person's repository. This person's pages function is very full, we only need to modify and cut down;
4. The video teaching is divided into personal version and multi-person version. The final personal version is not successful, the multi-person version is feasible, that is, the access address is very long;
5. I did not use the domain name I applied for, but used the one given by github.
6. The fork repository shows author names in many places, so we need to change them individually.
7. The comments of the article are the most troublesome. Originally, I applied for them from the third-party website of disqus, so I could only apply for one.
8. Because I don't know where to modify some wrong parts of the interface, I can only download the code to the local and let claude opus4.6 modify it;
